## 1. 架构概览

### 1.1 系统定位

平台作为 **MCP Client + 测评编排引擎**，不包含知识库存储与向量索引。知识库能力完全由远程 MCP Server 提供。

### 1.2 架构图

```

                             │ HTTPS / REST
                             ▼
┌─────────────────────────────────────────────────────────────────┐
│                      API Gateway / BFF                          │

         ▼                   ▼                   ▼
┌─────────────────┐ ┌─────────────────┐ ┌─────────────────┐
│  KB Service     │ │ Evaluation      │ │ Question Bank   │
│  MCP 挂载管理    │ │ Service         │ │ Service         │

         │                   │

         │  PostgreSQL     │ │  Redis          │

```

### 1.3 设计原则

| 原则 | 说明 |
|------|------|
| MCP 协议在后端 | 浏览器不直连 MCP；由后端 MCP Client 统一代理 |
| 响应归一化 | 任意 MCP Tool 返回经 Adapter 转为统一 `Chunk[]` |
| 测评可重放 | 保存 MCP 原始请求/响应，支持审计与调试 |
| 异步批处理 | 测评任务走队列，支持进度查询与取消 |
| 模块解耦 | 挂载、测评、评分、报告各自独立，便于扩展 |

## 3. 模块设计

### 3.1 KB Service（MCP 知识库服务）

**职责**：MCP 实例 CRUD、连接测试、Tool 发现、试检索。

**核心实体**：

```python
class MCPKnowledgeBase:
    id: str
    name: str
    endpoint: str
    auth_type: str          # api_key | bearer
    auth_secret_enc: str    # 加密存储
    retrieval_tool: str     # 如 kb_search
    tool_mapping: dict      # V1.1 参数/响应映射
    status: str             # connected | failed | pending
    last_tested_at: datetime
```

**关键接口**：

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/kb` | GET | 列表 |
| `/api/kb` | POST | 创建 |
| `/api/kb/{id}` | PUT | 更新 |
| `/api/kb/{id}` | DELETE | 删除 |
| `/api/kb/{id}/test-connection` | POST | 测试连接 |
| `/api/kb/{id}/discover-tools` | POST | 发现 Tools |
| `/api/kb/{id}/trial-search` | POST | 试检索 |

### 3.2 MCP Client

**职责**：与远程 MCP Server 通信，封装协议细节。

**能力**：

```
initialize(session) → 建立 MCP 会话
list_tools() → 获取 Tool 列表及 schema
call_tool(name, arguments) → 调用指定 Tool
health_check() → 可选 kb_health
```

**Adapter 层**：

```
Raw MCP Response → ChunkAdapter → List[Chunk]

Chunk:
  content: str
  source: str
  title: str | None
  score: float | None
```

**错误处理**：

| 错误 | 处理 |
|------|------|
| 连接超时 | 重试 1 次，标记 mcp_error |
| 鉴权失败 | 立即失败，提示检查密钥 |
| Tool 不存在 | 标记配置错误 |
| 响应解析失败 | mapping_error |

### 3.3 Evaluation Service（测评服务）

**职责**：测评任务生命周期管理、批量编排、进度跟踪。

**核心实体**：

```python
class EvaluationTask:
    id: str
    name: str
    kb_id: str
    question_bank_id: str
    mode: str               # retrieval_only | rag_full
    config: dict            # top_k, threshold, scoring_method
    status: str             # pending | running | completed | failed | cancelled
    progress: int           # 0-100
    started_at: datetime
    completed_at: datetime

class EvaluationResult:
    id: str
    task_id: str
    question_id: str
    mcp_request: dict
    mcp_response_raw: dict
    chunks: list
    generated_answer: str | None
    score: float
    passed: bool
    failure_type: str | None
    latency_ms: int
```

**执行流程**：

```
1. 创建任务 → 写入 DB，status=pending
2. 投递队列 → Worker 拉取
3. 加载题库 → 逐题循环：
   a. call MCP retrieval tool
   b. normalize chunks
   c. (optional) LLM generate
   d. score
   e. persist result
   f. update progress
4. 汇总 → 计算报告指标 → status=completed
```

### 3.4 Scoring Engine（评分引擎）

**职责**：单题评分、失败归因。

**策略**：

| 策略 | 适用模式 | 逻辑 |
|------|----------|------|
| `retrieval_match` | retrieval_only | 检查 source_ref 是否在 chunks 中；片段与期望答案语义相似度 |
| `llm_judge` | rag_full / retrieval_only | LLM 输出 score + passed + failure_type + reason |
| `semantic_similarity` | 通用 | embedding cosine similarity |

**LLM 裁判 Prompt 结构**：

```
输入：question, expected_answer, chunks, generated_answer(optional)
输出 JSON：{ score, passed, failure_type, reason }
```

### 3.5 Question Bank Service

**职责**：题库 CRUD、CSV 导入、内置题库管理。

### 3.6 Report Service

**职责**：聚合 EvaluationResult，生成报告视图数据。

**聚合指标**：

```
overall_score = avg(question_scores)
pass_rate = passed_count / total
avg_mcp_latency = avg(latency_ms)
failure_distribution = group_by(failure_type)
category_scores = group_by(category).avg(score)
```

## 5. 关键时序

### 5.1 挂载 MCP 知识库

```
User → Frontend → POST /api/kb
Backend → encrypt secret → save DB
User → POST /api/kb/{id}/test-connection
Backend → MCP Client.initialize(endpoint, auth)
Backend → MCP Client.list_tools()
Backend → return tools[] → Frontend
User → select tool → POST trial-search
Backend → MCP Client.call_tool(tool, {query, top_k})
Backend → Adapter → chunks[] → Frontend
User → save → status=connected
```

### 5.2 执行测评任务

```
User → POST /api/evaluations
Backend → create task, enqueue
Worker → for each question:
           MCP Client.call_tool(kb, query, top_k)
           → chunks
           → ScoringEngine.score()
           → save EvaluationResult
         → aggregate → mark completed
User → GET /api/evaluations/{id}/report
```

## 6. 安全设计

| 项 | 方案 |
|----|------|
| MCP 密钥 | AES 加密存 DB；API 返回脱敏（`sk-****abcd`） |
| 传输 | 全站 HTTPS；后端 → MCP 支持 TLS |
| 鉴权 | JWT Session；MVP 可环境变量单用户 |
| MCP 代理 | 后端校验 Endpoint 白名单（可选，防 SSRF） |
| 日志 | MCP 响应可截断存储，避免超大 payload |

## 10. 风险与对策

| 风险 | 影响 | 对策 |
|------|------|------|
| MCP 协议/传输差异 | 连接失败 | 优先支持 HTTP/SSE；文档明确传输要求 |
| MCP 服务不稳定 | 测评中断 | 单题超时跳过；重试；报告标注 mcp_error |
| Tool 返回格式不统一 | 解析失败 | MVP 标准契约；V1.1 映射配置 |
| LLM 裁判不稳定 | 分数波动 | 固定 prompt + temperature=0；可选人工抽检 |
| SSRF | 安全 | Endpoint 校验；内网地址黑名单 |
