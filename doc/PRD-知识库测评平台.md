# PRD — 知识库测评平台

**文档版本**：v0.1  
**更新日期**：2026-07-02  
**状态**：草案

---

## 1. 背景与目标

### 1.1 背景

企业在部署 RAG 知识库后，缺乏统一的、可量化的质量评估手段。现有方案多依赖人工抽检或各平台私有接口，难以对比迭代效果。

### 1.2 产品目标

构建一个 **Web 测评平台**，通过挂载远程 MCP 知识库服务，使用标准题库对知识库检索能力（及可选 RAG 全链路）进行自动化测评，输出可对比、可归因的测评报告。

### 1.3 非目标（MVP 不做）

- 本地文档上传、解析、向量化
- Dify / FastGPT 等专用连接器（统一走 MCP）
- 平台侧知识库索引管理
- 多租户 SaaS 计费体系

---

## 2. 目标用户

| 角色 | 诉求 | 核心操作 |
|------|------|----------|
| **知识库管理员** | 验证 MCP 知识库是否可用、检索是否准确 | 挂载 MCP、测试连接、发起测评 |
| **测评分析师** | 分析失败原因、导出报告、对比版本 | 查看报告、逐题分析、导出 |
| **研发/接入方** | 按规范实现 MCP 检索 Tool | 参考 MCP 接入规范 |

---

## 3. 产品定位

```
接入方式：远程 MCP 服务挂载（唯一方式）
测评对象：MCP Tool 返回的知识片段（检索测评）或 RAG 全链路回答（可选）
价值交付：量化得分 + 失败归因 + 可回归对比
```

---

## 4. 核心用户流程

```
挂载 MCP 知识库
    ↓
测试连接 & 试检索
    ↓
准备/导入测评题库
    ↓
创建测评任务（选 MCP KB + 题库 + 参数）
    ↓
批量执行（逐题调用 MCP Tool）
    ↓
查看测评报告（得分、归因、MCP 调用证据）
    ↓
（可选）导出报告 / 版本对比
```

## 5. 功能需求

### 5.1 MCP 知识库管理

#### 5.1.1 MCP 知识库列表

- 展示已挂载的 MCP 知识库实例
- 字段：名称、Endpoint、检索 Tool、连接状态、最近测试时间、操作
- 状态：`已连接` / `连接失败` / `待配置` / `已禁用`
- 操作：编辑、测试连接、试检索、删除、发起测评

#### 5.1.2 挂载向导（4 步）

| 步骤 | 内容 |
|------|------|
| Step 1 连接 | 名称、MCP Endpoint URL、鉴权方式（API Key / Bearer Token）、密钥 |
| Step 2 选择 Tool | 自动发现 MCP Tools，选择检索 Tool（推荐 `kb_search`） |
| Step 3 试检索 | 输入测试问题，预览返回片段（content / source / score） |
| Step 4 保存 | 确认配置摘要，保存为知识库实例 |

**验收标准**：
- 连接测试成功后可看到 Tool 列表
- 试检索可返回至少 0 条结果（空结果也视为调用成功）
- 密钥不在前端持久化明文展示

#### 5.1.3 Tool 映射（V1.1，MVP 可选）

当 MCP 服务未实现标准 `kb_search` 时，支持参数与响应字段映射：

- 入参映射：`query`、`top_k` → MCP Tool 实际字段名
- 响应映射：JSON 路径 → 统一 `Chunk[]` 结构

---

### 5.2 测评题库管理

#### 5.2.1 题库列表

- 内置标准题库（按场景分类，MVP 至少 1 套演示题库）
- 用户自定义题库（CSV 导入）

#### 5.2.2 题目字段

| 字段 | 必填 | 说明 |
|------|------|------|
| question | 是 | 测评问题 |
| expected_answer | 是 | 标准答案 |
| category | 否 | 类目（用于分组统计） |
| difficulty | 否 | 难度：easy / medium / hard |
| source_ref | 否 | 期望来源文档 ID 或标题（用于召回判定） |

#### 5.2.3 CSV 导入格式

```csv
question,expected_answer,category,difficulty,source_ref
退货政策是什么,7天内可无理由退货,售后政策,easy,售后手册第3章
```

---

### 5.3 测评任务

#### 5.3.1 创建测评

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| 任务名称 | 用户自定义 | — |
| MCP 知识库 | 选择已挂载实例 | — |
| 测评题库 | 选择内置或自定义题库 | — |
| 测评模式 | `retrieval_only` / `rag_full` | retrieval_only |
| top_k | 传给 MCP Tool 的检索条数 | 5 |
| 通过阈值 | 单题通过分数线 | 70 |
| 评分方式 | LLM 裁判 / 语义相似度 | LLM 裁判 |
| LLM 配置 | 生成/裁判模型（RAG 模式或 LLM 裁判时） | 系统默认 |

#### 5.3.2 测评执行

- 异步批量执行，支持进度展示
- 单题超时：默认 30s（可配置）
- 单题失败可跳过并继续，不阻断整批
- 支持取消进行中的任务

#### 5.3.3 任务状态

```
pending → running → completed
                 → failed
                 → cancelled
```

---

### 5.4 测评报告

#### 5.4.1 报告概览

- 综合得分（0–100）
- 通过率
- 平均 MCP 响应时延
- 失败题数及分类占比
- 按类目得分分布（若有 category）

#### 5.4.2 逐题详情

每题展示：

| 字段 | 说明 |
|------|------|
| 问题 | 题库原题 |
| 期望答案 | 标准答案 |
| 实际回答 | RAG 模式下 LLM 生成；检索模式下为空或 Top-1 片段摘要 |
| 检索片段 | MCP 返回的 chunks 列表 |
| MCP 调用证据 | Tool 名、请求参数、耗时、原始响应（可折叠） |
| 得分 | 单题得分 |
| 状态 | 通过 / 未通过 |
| 失败类型 | 见 5.4.3 |

#### 5.4.3 失败归因类型

| 类型 | 定义 |
|------|------|
| `mcp_error` | MCP 调用失败（网络、鉴权、Tool 报错） |
| `no_recall` | 调用成功但未召回相关片段，或 source_ref 未命中 |
| `wrong_answer` | 有召回但回答/匹配错误 |
| `hallucination` | 回答包含片段未支持的内容 |
| `incomplete` | 部分正确但不完整 |
| `mapping_error` | 响应解析/映射失败 |

#### 5.4.4 导出（V1.1）

- PDF 报告
- Excel 逐题明细

## 7. 非功能需求

| 类别 | 要求 |
|------|------|
| 性能 | 单题 MCP 调用 P95 < 5s；100 题测评任务 < 10min（依赖 MCP 性能） |
| 可用性 | MCP 连接测试响应 < 3s |
| 安全 | MCP 密钥后端加密存储；报告/API 不返回密钥明文 |
| 兼容性 | 桌面 Web，1280px+；中文界面 |
| 可观测 | 每次 MCP 调用全量日志（可采样存储） |

## 8. MVP 范围边界

### 8.1 MVP 包含

- [x] MCP 挂载（标准 `kb_search` Tool）
- [x] 连接测试、Tool 发现、试检索
- [x] CSV 题库导入 + 1 套内置题库
- [x] 检索测评模式（retrieval_only）
- [x] LLM 裁判评分
- [x] 测评报告（概览 + 逐题 + MCP 证据）
- [x] 基础登录

### 8.2 MVP 不包含

- [ ] Tool 灵活映射 UI
- [ ] RAG 全链路测评（rag_full）
- [ ] 版本对比
- [ ] PDF/Excel 导出
- [ ] 多租户 / RBAC

## 9. 成功指标

| 指标 | 目标 |
|------|------|
| MCP 挂载成功率 | > 95%（在符合规范的 MCP 服务上） |
| 单次测评任务完成率 | > 98%（允许单题失败） |
| 报告可读性 | 用户能在 5 分钟内定位 Top 3 失败原因 |
| 接入成本 | 接入方按 MCP 规范实现 1 个 Tool 即可对接 |

## 10. 开放问题

| # | 问题 | 建议 |
|---|------|------|
| 1 | MVP 是否必须支持 Tool 映射 | 建议 MVP 仅标准 `kb_search`，V1.1 加映射 |
| 2 | 检索测评的「通过」判定 | 有召回 + 片段与期望答案语义相似度 > 阈值 |
| 3 | 是否需要 RAG 全链路 | MVP 仅检索；V1.1 加 rag_full |
| 4 | 登录体系 | MVP 单用户 + 环境变量；后续 JWT + 多用户 |

## 附录 A：名词表

| 术语 | 定义 |
|------|------|
| MCP | Model Context Protocol，用于 AI 应用与外部工具/数据源的协议 |
| MCP 知识库 | 通过挂载远程 MCP 服务接入的知识库实例 |
| 检索 Tool | MCP 服务暴露的、用于按 query 检索知识片段的工具方法 |
| 检索测评 | 仅评估 MCP 检索结果质量，不经过 LLM 生成 |
| RAG 全链路测评 | 检索 + LLM 生成 + 与标准答案对比的完整测评 |
